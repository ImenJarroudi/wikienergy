Appliances¶

Appliance Trace¶

class disaggregator.appliance.ApplianceTrace(series, metadata)[source]¶

This class represents appliance traces.

Traces are power used by a single (or single set of) appliances sampled at a consistent rate.

get_sampling_rate()[source]¶

Returns a string representing the rate at which the series was sampled.

get_total_usage()[source]¶

Returns the total usage of this trace

print_trace()[source]¶

Prints a summary of the trace

Appliance Instance¶

class disaggregator.appliance.ApplianceInstance(traces, metadata)[source]¶

This class represents appliance instances, which may have multiple appliance traces.

Instances hold traces from a single instance of an appliance sampled at a consistent rate. Traces may or may not be consecutive.

concatenate_traces(how='strict')[source]¶

Takes its own list of traces and attempts to concatenate them.

Appliance Set¶

class disaggregator.appliance.ApplianceSet(instances, metadata)[source]¶

This class represents appliance sets, which contain a set of temporally aligned appliance instances.

Appliance sets are most frequently used as ground-truth for various algorithms, representing a particular home, building, or metered unit.

add_instances(instances)[source]¶

Adds the list of appliances to the appliance set.

add_to_dataframe(instances)[source]¶

Adds a new list of appliances to the dataframe.

generate_non_zero_set()[source]¶

Get all energy-consuming appliances (and drop instances with traces with all zeros)

generate_top_k_set(k)[source]¶

Get top k energy-consuming appliances

make_dataframe()[source]¶

Makes a new dataframe of the appliance instances. Throws an exception if if the appliance instances have traces that don’t align.

set_instances(instances)[source]¶

Replaces the old instances with the new list. Makes a new dataframe using those instances

Appliance Type¶

class disaggregator.appliance.ApplianceType(instances, metadata)[source]¶

This class represents appliance types, which contain a set of ApplianceInstances which share particular attributes, but which are not necessarily temporally aligned.

Appliance types are most frequently used to generate models of particular types of appliances.
